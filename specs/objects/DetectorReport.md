# DetectorReport

## Назначение

Отчёт detector harness по corpus и attack matrix.

## Канонические поля

- `report_id` — UUID строка
- `detector_version` — строка
- `corpus_id` — строка
- `attack_matrix_id` — строка
- `case_count` — целое >= 0
- `pass_rate` — число 0..1
- `confidence_floor` — число 0..1
- `notes` — массив строк

## Инварианты

- Pass rate ограничен диапазоном 0..1.
- Case count не отрицателен.
- `attack_matrix_id` обязателен, если report используется для claims уровня “100% идентификация”.

## Связи с другими объектами

- Опирается на `WatermarkEvidence`.
- Используется для критерия “100% идентификация” только в пределах утверждённой attack matrix.

## Каноническая сериализация

- JSON `snake_case`.

## Причины отклонения / ошибки

- отсутствует обязательное поле;
- нарушен инвариант;
- формат времени, enum или идентификатора невалиден;
- объект противоречит связанным контрактам;
- сериализация недетерминирована.

## Замечания по эволюции

- Обратимо-совместимые расширения добавляются новыми полями.
- Ломающие изменения требуют явного обновления `docs/PLAN.md`, `docs/STATUS.md` и связанных task-specific `AGENTS.md`.
- При изменении security semantics нужно дополнительно проверить trust boundary и audit model.
