# CompositionGraph

## Назначение

Итоговый граф composition после разрешения ASSETMAP/PKL/CPL/OV/VF.

## Канонические поля

- `composition_id` — UUID строка
- `source_packages` — массив package ids
- `resolved_reels` — массив Reel
- `warnings` — массив строк
- `missing_assets` — массив asset ids

## Инварианты

- Граф должен быть детерминирован для одного набора входных файлов.
- missing_assets и resolved_reels не противоречат друг другу.

## Связи с другими объектами

- Выдаётся dcp_probe.
- Кормит PlaybackTimeline.

## Каноническая сериализация

- JSON snake_case.
- Списки сортируются стабильно для CLI output.

## Причины отклонения / ошибки

- отсутствует обязательное поле;
- нарушен инвариант;
- формат времени, enum или идентификатора невалиден;
- объект противоречит связанным контрактам;
- сериализация недетерминирована.

## Замечания по эволюции

- Обратимо-совместимые расширения добавляются новыми полями.
- Ломающие изменения требуют явного обновления docs/PLAN.md, docs/STATUS.md и связанных task-specific AGENTS.md.
- При изменении security semantics нужно дополнительно проверить trust boundary и audit model.
